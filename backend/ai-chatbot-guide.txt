================================================================================
  HƯỚNG DẪN CỰC CHI TIẾT: TÍCH HỢP AI CHATBOT VÀO DỰ ÁN PORTFOLIO
================================================================================

  Dự án hiện tại: React (Vite) + Express + Sequelize + MySQL
  AI Provider:    OpenAI (GPT)
  Kết quả:        Nút chat ở góc phải dưới → mở chatbot hỏi đáp về bạn

================================================================================
  CẤU TRÚC THƯ MỤC SAU KHI THÊM CHATBOT
================================================================================

  backend/
  ├── config/
  │   ├── config.js          (đã có)
  │   ├── mailer.js          (đã có)
  │   └── openai.js          ← MỚI: OpenAI client
  ├── controller/
  │   ├── mailerController.js (đã có)
  │   └── chatbotController.js ← MỚI: xử lý chat request
  ├── service/
  │   ├── mailer_service.js   (đã có)
  │   └── chatbot_service.js  ← MỚI: gọi OpenAI API
  ├── routes/
  │   ├── contact.js          (đã có)
  │   └── chatbot.js          ← MỚI: route /api/chat
  ├── server.js               ← SỬA: thêm route chatbot
  └── .env                    ← SỬA: thêm OPENAI_API_KEY

  src/components/
  └── Chatbot/
      ├── Chatbot.jsx          ← MỚI: giao diện chatbot
      └── index.js             ← MỚI: barrel export

  KIẾN TRÚC LUỒNG DỮ LIỆU:
  User nhập message → React fetch POST /api/chat
                    → Express route → Controller → Service
                    → OpenAI API (gpt-4o-mini)
                    → Response trả về → Hiển thị trên chat window

  BƯỚC 5: TẠO FILE service/chatbot_service.js
========================================

  Đường dẫn: backend/service/chatbot_service.js

  ┌─────────────────────────────────────────────────────────┐
  │ const openai = require('../config/openai');              │
  │                                                         │
  │ // System prompt: "Dạy" AI về bạn                       │
  │ const SYSTEM_PROMPT = `                                 │
  │ You are a friendly assistant on Tan Phat Nguyen's       │
  │ portfolio website. Here is information about him:       │
  │                                                         │
  │ PERSONAL INFO:                                          │
  │ - Name: Tan Phat Nguyen (goes by Phat)                  │
  │ - Role: Back End Developer                              │
  │ - Location: Toronto, Canada                             │
  │ - Email: tannguyenntp2003@gmail.com                     │
  │                                                         │
  │ SKILLS:                                                 │
  │ - Languages: Java, Python, JavaScript, SQL              │
  │ - Backend: Spring Boot, Node.js, Express                │
  │ - Frontend: React, Tailwind CSS                         │
  │ - Database: MySQL, PostgreSQL, MongoDB                  │
  │ - Tools: Git, Docker                                    │
  │                                                         │
  │ RULES:                                                  │
  │ - Answer questions about Phat's skills and experience   │
  │ - Be friendly, professional, and concise                │
  │ - Keep answers to 2-3 sentences                         │
  │ - If asked something unrelated, politely redirect       │
  │ - Answer in the same language the user writes in        │
  │ - Do NOT make up information not listed above           │
  │ - If you don't know, say "I don't have that info,      │
  │   please contact Phat directly"                         │
  │ `;                                                      │
  │                                                         │
  │ async function chat(userMessage, history = []) {        │
  │   // Xây dựng mảng messages cho OpenAI                  │
  │   const messages = [                                    │
  │     { role: 'system', content: SYSTEM_PROMPT },         │
  │     ...history.slice(-10),  // Chỉ giữ 10 tin gần nhất │
  │     { role: 'user', content: userMessage },             │
  │   ];                                                    │
  │                                                         │
  │   const response = await openai.chat.completions.create({│
  │     model: 'gpt-4o-mini',                               │
  │     messages: messages,                                  │
  │     max_tokens: 300,                                     │
  │     temperature: 0.7,                                    │
  │   });                                                    │
  │                                                         │
  │   return response.choices[0].message.content;            │
  │ }                                                       │
  │                                                         │
  │ module.exports = { chat };                              │
  └─────────────────────────────────────────────────────────┘

  GIẢI THÍCH TỪNG PHẦN:

  ── SYSTEM_PROMPT ──
  → Đây là "bộ não" của chatbot
  → Cung cấp thông tin về bạn cho AI
  → AI sẽ trả lời dựa trên thông tin này
  → Bạn có thể thêm: projects, education, experience, hobbies
  → Càng chi tiết → AI càng trả lời chính xác
  → RULES giúp AI không bịa thông tin

  ── function chat(userMessage, history) ──

  Tham số:
    userMessage  = tin nhắn user vừa gửi (string)
    history      = lịch sử chat trước đó (array of {role, content})

  const messages = [
    { role: 'system', content: SYSTEM_PROMPT },
    ...history.slice(-10),
    { role: 'user', content: userMessage },
  ];

  → Cấu trúc messages cho OpenAI API:
    - system: Hướng dẫn AI cách hành xử (SYSTEM_PROMPT)
    - history: Các tin nhắn trước (để AI nhớ ngữ cảnh)
    - user: Tin nhắn mới nhất
  → history.slice(-10): Chỉ giữ 10 tin gần nhất
    (tiết kiệm tokens = tiết kiệm tiền)

  openai.chat.completions.create({...})
  → Gọi API tạo chat completion

  Các tham số:
    model: 'gpt-4o-mini'
    → Model AI để dùng
    → gpt-4o-mini: rẻ nhất ($0.00015/1K tokens), nhanh, đủ tốt
    → Các lựa chọn khác:
      gpt-3.5-turbo: $0.002/1K tokens, nhanh
      gpt-4o:        $0.005/1K tokens, thông minh hơn
      gpt-4:         $0.03/1K tokens, chậm nhất

    max_tokens: 300
    → Giới hạn độ dài response (~200 từ)
    → Càng cao → response dài hơn + tốn tiền hơn

    temperature: 0.7
    → Độ "sáng tạo" của AI (0.0 → 2.0)
    → 0.0 = cực kỳ chính xác, lặp lại câu trả lời
    → 0.7 = cân bằng giữa chính xác và tự nhiên
    → 1.0 = rất sáng tạo, có thể bịa
    → Khuyên dùng: 0.5-0.7 cho chatbot portfolio

  response.choices[0].message.content
  → Lấy nội dung text từ response
  → choices[0] = câu trả lời đầu tiên (thường chỉ có 1)
  → .message.content = text string


========================================
  BƯỚC 6: TẠO FILE controller/chatbotController.js
========================================

  Đường dẫn: backend/controller/chatbotController.js

  ┌─────────────────────────────────────────────────────────┐
  │ const { chat } = require('../service/chatbot_service'); │
  │                                                         │
  │ const handleChat = async (req, res) => {                │
  │   const { message, history } = req.body;                │
  │                                                         │
  │   // Validate: phải có message                          │
  │   if (!message || typeof message !== 'string') {        │
  │     return res.status(400).json({                       │
  │       error: 'Message is required'                      │
  │     });                                                 │
  │   }                                                     │
  │                                                         │
  │   // Validate: message không quá dài (chống spam)       │
  │   if (message.length > 500) {                           │
  │     return res.status(400).json({                       │
  │       error: 'Message too long (max 500 characters)'    │
  │     });                                                 │
  │   }                                                     │
  │                                                         │
  │   try {                                                 │
  │     const reply = await chat(message, history || []);   │
  │     res.json({ reply });                                │
  │   } catch (err) {                                       │
  │     console.error('OpenAI Error:', err.message);        │
  │     res.status(500).json({                              │
  │       error: 'AI is not available right now'            │
  │     });                                                 │
  │   }                                                     │
  │ };                                                      │
  │                                                         │
  │ module.exports = { handleChat };                        │
  └─────────────────────────────────────────────────────────┘

  GIẢI THÍCH TỪNG PHẦN:

  const { message, history } = req.body;
  → Lấy 2 field từ body:
    message = tin nhắn user (string)
    history = lịch sử chat (array, optional)

  Validation 1: if (!message || typeof message !== 'string')
  → Kiểm tra message có tồn tại và đúng kiểu string
  → Trả 400 nếu thiếu

  Validation 2: if (message.length > 500)
  → Giới hạn 500 ký tự
  → Chống spam + tiết kiệm tokens
  → Bạn có thể chỉnh số này tùy ý

  const reply = await chat(message, history || []);
  → Gọi service, truyền history hoặc array rỗng nếu không có
  → await vì gọi API bên ngoài (async)

  res.json({ reply });
  → Trả response: { "reply": "AI's answer here..." }

  catch: res.status(500).json({ error: '...' })
  → Nếu OpenAI API lỗi (hết credit, rate limit, server down)
  → Trả message thân thiện cho user
  → Log lỗi thật ra console để debug

    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages])

  → Mỗi khi messages thay đổi (có tin mới)
  → Scroll mượt xuống cuối cùng
  → ?. (optional chaining) để tránh lỗi khi ref chưa mount

  ── sendMessage() ──

  if (!input.trim() || isLoading) return
  → Không gửi nếu input rỗng hoặc đang loading

  const userMsg = { role: 'user', content: input }
  → Tạo object tin nhắn user

  setMessages(newMessages)
  → Thêm tin nhắn user vào UI ngay lập tức (UX tốt)

  setInput('')
  → Xóa ô input

  fetch('http://localhost:5000/api/chat', { body: { message, history } })
  → Gọi backend API
  → Gửi tin nhắn hiện tại + lịch sử chat
  → Backend forward tới OpenAI → nhận reply

  if (data.reply) → setMessages thêm tin nhắn AI
  → Hiển thị câu trả lời AI

  catch → hiện "Cannot connect to server."
  → Khi backend không chạy hoặc lỗi mạng

  ── handleKeyDown() ──

  if (e.key === 'Enter' && !e.shiftKey)
  → Nhấn Enter để gửi
  → Shift+Enter để xuống dòng (không gửi)

  ── UI LAYOUT ──

  fixed bottom-6 right-6 z-50
  → Nút chat cố định ở góc phải dưới
  → z-50 để luôn hiện trên các element khác

  Nút: w-14 h-14 rounded-full
  → Hình tròn, 56px, màu xanh
  → Hiện 💬 khi đóng, ✕ khi mở

  Cửa sổ: absolute bottom-16 right-0 w-80
  → Mở phía trên nút chat
  → 320px rộng (sm:w-96 = 384px trên màn lớn)
  → max-h-[500px] giới hạn chiều cao

  Messages: user (bg-blue-500, phải) / bot (bg-gray-100, trái)
  → Giống giao diện iMessage
